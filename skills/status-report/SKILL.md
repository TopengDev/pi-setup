# Weekly Client Status Report Generator

Generate a professional, client-ready status report by analyzing the project's real state — git history, issues, codebase health, and project artifacts.

## 1. Determine the Report Period

Default to the last 7 days ending today. If a date range is specified, use that.

## 2. Gather Project Identity

Determine project name from: `CLAUDE.md`, `package.json` name, `go.mod` module path, `Cargo.toml`, git remote URL, or directory name.

## 3. Gather Data

### 3a. Git History
```bash
git log --since="$SINCE" --until="$UNTIL" --oneline --no-merges
git log --since="$SINCE" --until="$UNTIL" --stat --no-merges
git log --since="$SINCE" --until="$UNTIL" --format='%aN' | sort -u
git diff --shortstat $(git rev-list -1 --before="$SINCE" HEAD) HEAD
```

### 3b. GitHub Issues & PRs (if `gh` CLI available)
```bash
gh pr list --state merged --search "merged:>=$SINCE" --limit 50 --json number,title,author,mergedAt
gh pr list --state open --limit 20 --json number,title,author,createdAt,labels
gh issue list --state open --limit 30 --json number,title,labels,createdAt
gh issue list --state open --label bug --limit 20 --json number,title,createdAt
```

### 3c. Codebase Health
Count TODO/FIXME/HACK comments across source files using grep.

### 3d. Test & Build Status
Check for recent CI results, test coverage reports, build status.

### 3e. Project Artifacts
Read: `TODO.md`, `ROADMAP.md`, `CHANGELOG.md`, `MILESTONES.md` if they exist.

## 4. Analyze & Synthesize

- **Group commits by area** — translate git activity into human-readable accomplishments
- **Infer project health** — velocity, open bug count, stale issues, failing CI
- **Identify blockers** — issues labeled "blocked", PRs with unresolved reviews
- **Determine what's next** — from open PRs, milestone labels, ROADMAP.md

## 5. Write the Report

Generate to `./status-report-$UNTIL.md`:


## 6. Report Rules

- **Client-friendly language** — no jargon. "user login system" not "auth middleware refactor."
- **Honest assessment** — if progress was slow, say so.
- **Specific, not vague** — "Completed checkout with Stripe integration" not "Made progress on payments."
- **No fabrication** — only report what can be verified from data gathered.
- **Always keep**: Header, Health Dashboard, Completed, In Progress, Metrics sections.

## 7. After Writing

1. Tell user where the report is
2. Offer to adjust tone, add/remove sections, or generate executive summary version
